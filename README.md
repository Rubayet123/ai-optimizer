# 🧠 Install RTK

## 1. Download Installer

[install-rtk.bat](https://github.com/Rubayet123/ai-optimizer/tree/main/rtk/install-rtk.bat)

## 2. Run as Administrator

Right-click → **Run as Administrator**

## 3. Verify

```bash
rtk --version
```

# 🧩 Install ECC

> Install ECC inside each project folder.

## 1. Open Project

```bash
cd path/to/project
```

## 2. Clone ECC

```bash
git clone https://github.com/affaan-m/everything-claude-code.git
```

## 3. Install Dependencies

```bash
cd everything-claude-code
npm install
```

## 4. Run Installer

```bash
cd ..
./everything-claude-code/install.sh target anti-gravity typescript
```

*(Note: Replace `typescript` with `python` or your respective language profile if configuring a dedicated backend service).*

## 5: Start a Fresh Session / Restart Antigravity

Reopen your project workspace inside Google Antigravity or launch a new conversation. Antigravity will automatically index the metadata descriptions inside your new local `.agent/skills/` and `.agent/rules/` directories.

## 🏗️ Production Web App Lifecycle Matrix

Once installed, you don't need to memorize every command. Thanks to progressive disclosure, Antigravity matches your prompts to these underlying skills automatically. You can also explicitly call them using the trigger phrases below.

### 1. The Planning & Architecture Stage

*Focuses on system design, library scoping, and workflow mapping before writing functional files.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/code-architect` | Maps multi-file technical blueprints, architectural interfaces, and construction order. | `"Design the data flow and system architecture for our new MCQ analytics dashboard."` |
| `/plan` | Builds precise operational state-machines, code logic patterns, and layout maps. | `"/plan map out the state validation steps for user quiz submissions."` |
| `/market-research` | Evaluates public tech stacks, third-party libraries, and production SDKs. | `"Find the best Stripe subscription library for a Next.js React app."` |
| `/strategic-compaction` | Squashes active chat histories to optimize active context window token speeds. | `"/compact condense our workspace chat history to free up active context tokens."` |

### 2. The Frontend Design & UI/UX Stage

*Handles semantic view construction, design system consistency, reactive state tracking, and compliance.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/ui-scaffold` | Deploys responsive layout frameworks, utility classes, and view skeleton components. | `"Scaffold a responsive grid system for our quiz category cards using Tailwind CSS."` |
| `/state-manager` | Configures global client storage providers, event dispatches, and responsive tabs. | `"Configure a global React Context store to manage user auth state across tabs."` |
| `/accessibility` | Validates DOM tree composition against global accessibility frameworks (WCAG 2.2). | `"Audit this quiz submission form for full WCAG 2.2 accessibility compliance."` |
| `/responsive-check` | Audits viewport layout shift parameters, CSS breakpoints, and element wraps. | `"Check this landing page layout for responsive styling bugs down to mobile viewports."` |

### 3. The Backend & Database Infrastructure Stage

*Secures endpoints, wires data persistency layers, configures edge optimizations, and guards APIs.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/api-builder` | Engineers structured API endpoints with type protection and payload serialization. | `"Build a Node.js API endpoint to securely handle and validate payment webhooks."` |
| `/db-migrator` | Generates declarative database schemas, structural indexes, and live seeds. | `"Create a Prisma migration file to add a score column to our database schema."` |
| `/cache-optimizer` | Sets up edge caching layer routers, KV store mappings, and cache header strings. | `"Implement a cache-first routing structure using Cloudflare D1 and Hono.js keys."` |
| `/rate-limiter` | Embeds security throttles to neutralize scrapers or unauthorized endpoint hammering. | `"Add an API rate-limiter middleware targeting the user login and token route."` |

### 4. The Search Engine Optimization (SEO) Stage

*Ensures high indexability, rich crawler interpretation, link canonicalization, and swift page rendering.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/metadata-injector` | Programs structured JSON-LD schema marks, OpenGraph tags, and contextual page meta values. | `"Inject programmatic SEO meta headers and OpenGraph tags into this dynamic route."` |
| `/sitemap-generator` | Programs dynamic XML chunk indexes to scale content maps across thousands of pages. | `"Write a script to programmatically split and generate sitemaps for our MCQ pages."` |
| `/vitals-audit` | Evaluates Core Web Vitals targets to maximize ranking weight potential. | `"Analyze our landing page build for performance improvements to maximize SEO rankings."` |
| `/canonical-enforcer` | Normalizes loose trailing URL parameters and structural root query routes. | `"Configure canonical URL routing handling rules inside our edge middleware script."` |

### 5. The Implementation & Automated Testing Stage

*Drives production feature engineering with test verification and automated self-repair mechanisms.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/tdd` | Forces a safe Test-Driven Development workflow (failing test ➔ implementation ➔ green). | `"Write a failing test for our email pattern validator utility function first."` |
| `/build-error-resolver` | Intercepts active runtime stack traces to deploy minimal safe adjustments. | `"Our Vite production bundle failed with a typing error. Locate and resolve it."` |
| `/mock-seeder` | Generates deterministic mock JSON records to safely benchmark performance scopes. | `"Generate a mock JSON payload containing realistic questions to populate our test DB."` |

### 6. The Quality Assurance & Security Stage

*Runs strict security hooks, automated double-check loops, and peer-review code evaluations.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/code-review` | Inspects workspace code modifications against strict edge-case engineering checklists. | `"Review these local code changes for errors or edge-cases like a Senior Engineer."` |
| `agent-shield` | Hooks into active tasks to prevent programmatic leaks of raw access tokens or key strings. | **Automatic Behavior Hook:** Intercepts pipeline commands if an uncommitted key is detected. |
| `/verification-loops` | Locks the agent in a recursive test-and-repair script until zero syntax errors remain. | `"Execute a validation verification loop over our migration handler logic."` |
| `/agent-eval` | Runs localized side-by-side benchmarking profiles across available inference weights. | `"Compare Gemini Pro and Flash models for refactoring this heavy text process."` |

### 7. The Shipping & Deployment Stage

*Normalizes source histories, automates atomic branch logic, and packages production assets.*

| Skill / Command | Technical Workspace Purpose | Intent / Trigger Phrase Example |
| --- | --- | --- |
| `/conventional-commits` | Evaluates active multi-file code diff records to formulate clean, semantic commit titles. | `"Analyze my uncommitted modifications and log them using conventional structure."` |
| `/batch` | Distributes complex structural modifications atomically across decoupled project trees. | `"/batch propagate the updated user metadata object properties across project modules."` |
