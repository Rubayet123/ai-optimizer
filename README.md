## 🧠 1. RTK (Rust Token Killer)

Download the **install-rtk.bat** file > open with administration > install RTK
https://github.com/Rubayet123/ai-optimizer/tree/main/rtk/install-rtk.bat

RTK acts as a "filter" between your terminal commands and the AI. Instead of sending a 500-line error log to the LLM (which costs money and eats up context), RTK identifies the "signal" and removes the "noise."

## ⚡ Useful Commands (Cheatsheet)

### 📊 Monitoring Savings

| Command | Description |
| --- | --- |
| `rtk gain` | **The Scoreboard.** Shows total tokens and dollars saved. |
| `rtk gain --graph` | Displays an ASCII graph of savings over the last 30 days. |
| `rtk gain --history` | Lists recent commands and their specific compression ratios. |

### 🛠 Manual Usage (CLI)

While your AI Agent uses these automatically, you can run them manually to keep your own terminal clean:

* **Git Optimization:**
* `rtk git status` — Returns a ultra-compact status.
* `rtk git diff` — Condenses changes to logic only (ignores whitespace/boilerplate).


* **Development & Tests:**
* `rtk npm test` / `rtk cargo test` — Hides passing tests; only shows the AI why a test *failed*.
* `rtk tsc` — Groups TypeScript errors by file for easier debugging.


* **Smart Discovery:**
* `rtk ls .` — Generates a token-optimized directory tree.
* `rtk read <file>` — Reads a file but strips comments/whitespace to save context.

## 🔧 Maintenance & Troubleshooting

* **Update Global Rules:** If Antigravity stops recognizing RTK, run:

```powershell
    rtk init -g --agent antigravity
```

*   **Update Binary:** Simply run the `setup.bat` again; it will fetch the latest release from GitHub.
*   **Verification:** Run `rtk --version` to ensure the system PATH is correctly configured.

# 🚀 Ultimate Guide: Building Production-Ready Web Apps with Antigravity & Everything Claude Code (ECC)

This tutorial shows you how to legitimately supercharge the free tier of **Google Antigravity** using the **Everything Claude Code (ECC)** toolkit. By embedding these reusable agentic workflows directly into your local project workspace, you transform your AI assistant from a simple code wrapper into a structured, multi-stage engineering pipeline.

---

## 🛠️ Step-by-Step Installation Tutorial

Always install the ECC toolkit directly inside the **root directory** of your specific project workspace (e.g., `projects/website1`) to keep project rules and frameworks safely decoupled.

### Step 1: Open Your Project Workspace
Launch your terminal and navigate to your active website or web app directory:
```bash
cd path/to/projects/website1

```

### Step 2: Clone the ECC Repository Locally

Download the toolkit directly into your project root folder:

```bash
git clone [https://github.com/affaan-m/everything-claude-code.git](https://github.com/affaan-m/everything-claude-code.git)

```

### Step 3: Install Core Dependencies

Navigate into the newly cloned directory to build the required Node.js package scripts:

```bash
cd everything-claude-code
npm install

```

### Step 4: Run the Target Configuration Script

Move back up to your project root and execute the installer script, passing `anti-gravity` as your target platform:

```bash
cd ..
./everything-claude-code/install.sh target anti-gravity typescript

```

*(Note: Replace `typescript` with `python` or your respective language profile if configuring a dedicated backend service).*

### Step 5: Start a Fresh Session

Reopen your project workspace inside Google Antigravity or launch a new conversation. Antigravity will automatically index the metadata descriptions inside your new local `.agent/skills/` and `.agent/rules/` directories.
